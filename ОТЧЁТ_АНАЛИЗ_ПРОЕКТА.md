# Отчёт по анализу проекта Floor Haven (ПолМастер)

**Дата анализа:** 29 января 2026  
**Проект:** Сайт компании по продаже напольных покрытий «ПолМастер»

---

## 1. Общее описание

**Floor Haven** — одностраничный маркетинговый сайт + каталог напольных покрытий для компании «ПолМастер». Реализован как SPA на React с маршрутизацией, каталогом с фильтрами, формами заявок и информационными страницами. Проект создан на базе шаблона Lovable (Vite + React + shadcn/ui).

**Целевая аудитория:** B2C — частные клиенты и дизайнеры интерьеров, ищущие напольные покрытия в Москве и регионах.

---

- Порт dev-сервера: 8080, HMR overlay отключён.

## 3. Структура проекта

```
floor-haven/
├── public/              # Статика (favicon, robots.txt, placeholder)
```

Логика приложения сосредоточена в `src`: страницы, компоненты, данные и утилиты. UI-библиотека shadcn вынесена в `components/ui`.

## 4. Маршрутизация и страницы

| Путь | Страница | Описание |
|------|----------|----------|
| `/` | Index | Главная: Hero, преимущества, о компании, товары, отзывы, форма заявки |
| `/catalog` | Catalog | Каталог с фильтрами и сеткой карточек товаров |
| `/about` | AboutCompany | О компании: история (таймлайн), команда, CTA |
| `/faq` | FAQ | Вопросы-ответы (Accordion) |
| `/contacts` | Contacts | Контакты, карта, реквизиты, соцсети |
| `*` | NotFound | 404 с логированием в консоль |

Роутинг настроен в `App.tsx` через `BrowserRouter` и `Routes`. Общие блоки (Header/Footer) повторяются на каждой странице, кроме главной (там один общий layout).

## 5. Ключевые компоненты

### 5.1 Header
- Фиксированный, скрывается при скролле вниз (после 100px), появляется при скролле вверх.
- При скролле > 50px — фон с blur и тенью.
- Логотип «ПолМастер», навигация (Каталог, О компании, Вопросы, Контакты), CTA «Оставить заявку» (скролл к `#contact`).
- Мобильное меню (бургер) с теми же ссылками.

### 5.2 Hero
- Двухколоночный блок: заголовок, описание, кнопки «Получить консультацию» / «Смотреть каталог», блок статистики (15+ лет, 5000+ клиентов, 50+ брендов).
- Справа — изображение с плавающей карточкой «Бесплатный замер».

### 5.3 Каталог (Catalog)
- **Данные:** товары из `src/data/products.ts` (12 позиций: ламинат, паркет, винил, кварц-винил, инженерная доска, пробка).
- **Фильтры (ProductFilters):** категория, бренд, класс износостойкости, влагостойкость, толщина (мм), цена (слайдер 500–6000 ₽), наличие. Секции сворачиваются/разворачиваются.
- **Поведение:** фильтрация через `useMemo` по всем полям; на мобильных фильтры в `Sheet` (боковая панель), счётчик активных фильтров на кнопке.
- **Склонение:** «N товар / товара / товаров» через функцию `getProductWord`.

### 5.4 ProductCard
- Картинка, бейджи «В наличии»/«Под заказ», цена (₽/м²), бренд, название, категория, класс износостойкости, толщина, влагостойкость. Кнопка «Подробнее» без перехода (пока заглушка).

### 5.5 ContactForm
- Поля: имя, телефон. Отправка имитируется задержкой 1 с; после «успеха» показывается экран «Заявка отправлена» с возможностью отправить ещё раз. Реальных запросов на бэкенд нет.

### 5.6 Остальные блоки
- **Advantages:** 4 карточки (доставка, гарантия, цены, поддержка).
- **About:** текст о компании, изображение шоурума, два блока статистики (500+ видов, 98% клиентов).
- **Products:** 6 категорий с картинками и ценами «от …»; кнопка «Подробнее» без перехода.
- **Reviews:** карусель из 3 отзывов с фото, рейтинг 5 звёзд, навигация стрелками и точками.

## 6. Данные и типы

**Файл:** `src/data/products.ts`

- **Типы:** `ProductCategory`, `WearClass`, `Product` (id, name, category, brand, price, inStock, image, wearClass, waterResistant, thickness).
- **Массивы:** `products` (12 товаров), `categories`, `brands`, `wearClasses`, `thicknesses`.
- Изображения каталога импортируются напрямую (Vite). Остальной контент (тексты, отзывы, команда, FAQ, таймлайн) захардкожен в компонентах и страницах.

**Состояние:** только локальный state (useState) в страницах и компонентах; глобального стора (Redux/Zustand и т.п.) нет. React Query подключён в `App.tsx`, но запросов к API нет — данные статичные.

## 7. Стили и дизайн-система

- **Шрифт:** Inter (Google Fonts).
- **Цвета:** CSS-переменные в `index.css` (light/dark): background, foreground, primary (синий), secondary, muted, accent, destructive, border, input, ring, card, popover. Поддержка тёмной темы объявлена, переключателя в UI нет.
- **Компонентные классы:** `container-custom`, `btn-primary`, `btn-outline`, `card-elevated`, `section-padding`, `heading-xl/lg/md`, `text-body`, `text-small`, `icon-box`, `input-field`, `gradient-text`, `animate-fade-in`, `animate-slide-up`.
- **Tailwind:** используется повсеместно; конфиг расширяет тему (цвета, радиус, анимации accordion). Плагин `tailwindcss-animate` подключён.

Единый визуальный стиль: карточки, кнопки, отступы и типографика выдержаны.

---

## 8. Качество кода и настройки TypeScript

- **Строгость:** в `tsconfig` отключены `strictNullChecks`, `noImplicitAny`, `noUnusedLocals`, `noUnusedParameters` — типизация ослаблена для быстрой разработки.
- **Компоненты:** функциональные, на хуках; пропсы типизированы (интерфейсы для ProductCard, ProductFilters, FilterState).
- **Тесты:** один примерный тест в `src/test/example.test.ts`; бизнес-логика и компоненты тестами не покрыты.
- **Доступность:** семантика в целом нормальная (секции, заголовки, формы с label). Кнопки-иконки без подписей (например, в Reviews) лучше дополнить `aria-label`.

## 9. Замечания и рекомендации

### Критичные / важные
1. **Форма заявки** не отправляет данные на сервер — нужен реальный API или интеграция с CRM/почтой.
2. **Кнопки «Подробнее»** в каталоге и на главной никуда не ведут — стоит добавить страницу товара или модальное окно с деталями.
3. **Footer:** ссылки «Навигация» ведут на `#` — заменить на реальные маршруты (`/`, `/about`, `/catalog` и т.д.) или якоря.
4. **FAQ:** ссылка «Связаться с нами» ведёт на `/contacts` через `<a href>`. Лучше использовать `<Link>` из React Router для консистентности SPA.

### Улучшения
5. **React Query** не используется при статичных данных — либо убрать зависимость, либо позже использовать для загрузки каталога/заявок с бэкенда.
6. **Дублирование данных:** список товаров на главной (Products) задан отдельно от `data/products.ts` — при изменении ассортимента легко рассинхрон; лучше брать категории/товары из одного источника.
7. **404:** текст на английском («Return to Home») — для русскоязычного сайта уместнее «На главную» и `<Link to="/">`.
8. **Контакты и соцсети:** ссылки VK/Telegram общие (vk.com, t.me) — заменить на реальные при наличии.
9. **Тесты:** добавить хотя бы тесты на фильтрацию каталога и ключевые компоненты (Header, ContactForm, ProductFilters).
10. **SEO:** проверить мета-теги, title и описание для каждой страницы (например, через React Helmet или аналог).

### Плюсы
- Понятная структура папок и разделение по страницам/компонентам.
- Удобная фильтрация каталога и адаптив (в т.ч. мобильные фильтры в Sheet).
- Единая дизайн-система (Tailwind + свои классы + shadcn).
- Корректная маршрутизация и обработка 404.
- Использование типов и интерфейсов для продуктов и фильтров.
